GUIA PARA RODAR O BOT NO COLAB E COLOCAR NO SERVIDOR
=====================================================

Este guia explica como colocar o bot no seu servidor Discord e como executar o projeto pelo Google Colab.

O notebook principal criado para isso fica em:

notebooks/bot_discord_colab_runner.ipynb


1. O QUE VOCE PRECISA
---------------------

Voce precisa de:

- Uma conta Discord.
- Permissao para adicionar bots no servidor.
- Um bot criado no Discord Developer Portal.
- O token do bot.
- O repositorio publicado no GitHub.
- Google Colab.

Para esta fase o NGrok já é obrigatório pois o web painel serve de workaround para burlar as limitações de áudio base do Discord.


2. CRIAR O BOT NO DISCORD DEVELOPER PORTAL
------------------------------------------

1. Acesse:

   https://discord.com/developers/applications

2. Clique em "New Application".

3. Escolha um nome para a aplicacao.

4. Entre na aplicacao criada.

5. Va em "Bot".

6. Clique para criar/adicionar o bot, se ele ainda nao existir.

7. Copie o token do bot.

   Esse token sera usado no Colab como DISCORD_TOKEN.

Importante:

- Nao coloque esse token no GitHub.
- Nao mande esse token para outras pessoas.
- Se vazar, clique em "Reset Token" no Developer Portal.


3. CONFIGURAR O BOT NO DEVELOPER PORTAL
---------------------------------------

Na aba "Bot":

1. Deixe o bot ativo.

2. Nao precisa ativar "Message Content Intent" agora.

3. Para a fase atual, o bot usa slash commands e voz.

4. Guarde o token para usar no notebook.


4. CONVIDAR O BOT PARA O SEU SERVIDOR
-------------------------------------

No Discord Developer Portal:

1. Va em "OAuth2".

2. Va em "URL Generator".

3. Em "Scopes", marque:

   - bot
   - applications.commands

4. Em "Bot Permissions", marque pelo menos:

   - View Channels
   - Send Messages
   - Connect
   - Speak
   - Use Voice Activity

5. Copie a URL gerada.

6. Abra a URL no navegador.

7. Escolha seu servidor.

8. Autorize o bot.

Se preferir montar a URL manualmente, voce precisa do Client ID da aplicacao:

https://discord.com/oauth2/authorize?client_id=SEU_CLIENT_ID&scope=bot%20applications.commands&permissions=36703232

Troque SEU_CLIENT_ID pelo Application ID/Client ID do seu bot.


5. PREPARAR O REPOSITORIO NO GITHUB
-----------------------------------

Antes de abrir o notebook, envie este repositorio para o GitHub.

O notebook vai clonar o repositorio usando a URL:

https://github.com/SEU_USUARIO/bot-discord-colab.git

Se o repositorio for privado, voce precisa usar uma forma de acesso autenticada no Colab.

Para repositorio publico, basta a URL normal.

12. PROBLEMAS COMUNS
--------------------

Erro: DISCORD_TOKEN nao configurado

Solucao:

- Rode a celula de variaveis secretas.
- Cole o token do bot.


Erro: comandos slash nao aparecem

Solucao:

- Confirme que o bot foi convidado com applications.commands.
- Espere alguns segundos.
- Reabra o Discord ou troque de canal.


Erro: bot nao entra na call

Solucao:

- Entre em uma call antes de usar /join.
- Confirme que o bot tem permissao Connect.
- Confirme que o bot consegue ver o canal.


Erro: bot entra mas nao toca audio

Solucao:

- Confirme que ffmpeg foi instalado no Colab.
- Confirme que o bot tem permissao Speak.
- Rode a celula que cria /content/test_audio.wav.
- Use /play_test_audio path:/content/test_audio.wav.


Erro: gravacao nao salva audio

Solucao:

- Confirme que o bot esta na call.
- Use /record_test seconds:5.
- Fale durante a gravacao.
- Verifique /content/discord_recordings.
